# 安全增强系统

## 安全架构层次

| 层级 | 组件 | 职责 |
|------|------|------|
| 客户端层 | RainbowKit + Wagmi | 钱包连接、Cookie 持久化 |
| API 层 | 聊天/工具/验证/删除 API | 请求验证、速率限制 |
| 业务逻辑层 | LLM Provider + 工具执行器 | 数据验证、工具参数校验 |
| 数据层 | Supabase + RLS | 行级安全策略、服务端角色 |

---

## 身份认证

- **方式**：基于钱包的去中心化认证（RainbowKit）
- **状态管理**：Wagmi 管理连接状态，Cookie 支持 SSR 恢复
- **地址验证**：所有 API 使用 `isValidEthereumAddress` 验证格式

---

## 数据库安全策略（RLS）

| 表 | SELECT | INSERT | UPDATE | DELETE |
|----|--------|--------|--------|--------|
| conversations | 允许所有 | 仅认证用户 | 应用层验证 | 仅服务端 API |
| messages | 允许所有 | 仅认证用户 | 应用层验证 | 仅服务端 API（级联） |
| transfer_cards | 允许所有 | 仅认证用户 | 应用层验证 | 应用层验证 |

---

## 对话安全管理

**双重验证机制**：
1. `POST /api/supabase/verify-ownership` -- 验证对话所有权
2. `POST /api/supabase/delete-conversation` -- 服务端执行删除

**实现细节**：
- 使用 service_role 密钥绕过 RLS 执行删除
- DELETE 条件同时匹配 conversationId + walletAddress
- 外键 ON DELETE CASCADE 自动清理 messages

---

## API 安全措施

| 措施 | 实现方式 |
|------|----------|
| 输入验证 | 钱包地址格式、工具参数、chainId 枚举 |
| 速率限制 | Nginx 层实现 |
| 错误处理 | 区分配置错误(503)和运行时错误(500)，信息脱敏 |
| 流式安全 | SSE 响应安全处理，支持 AbortController 中止 |
| 传输加密 | HTTPS（Nginx + Let's Encrypt） |

---

## 部署安全配置

Nginx 安全 Headers：
- `X-Frame-Options: DENY`
- `X-Content-Type-Options: nosniff`
- `X-XSS-Protection: 1; mode=block`
- `Strict-Transport-Security` (HSTS)

---

## 生产环境 RLS 升级步骤

1. 获取 Supabase Service Role Key
2. 设置环境变量 `SUPABASE_SERVICE_ROLE_KEY`
3. 执行 `supabase/migrations/upgrade_production_rls.sql`
4. 验证：前端直接 DELETE 被拦截，服务端 API 删除正常
