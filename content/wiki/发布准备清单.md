# 文档发布准备清单

> 用于检查文档体系是否达到发布标准

---

## 发布前检查项

### 1. 文档结构
- [x] 目录结构清晰，按数字前缀排序
- [x] 每个分类有明确的职责边界
- [x] 文档命名规范统一
- [x] 创建总入口 README.md
- [x] 删除或备份重复和过时的文档

### 2. 内容质量
- [ ] 所有文档内容与项目现状一致
- [ ] 文档中的代码示例可运行
- [ ] 文档中的文件路径和链接正确

### 3. 格式一致性
- [x] Markdown 格式统一
- [x] 标题层级一致（# / ## / ###）
- [x] 代码块带语言标识
- [x] UTF-8 编码，换行符一致

### 4. 完整性
- [x] 核心文档齐全：项目介绍、快速上手、AI Agent 学习指南、技能系统、部署指南、API 参考、故障排除
- [x] README.md 包含完整文档导航和按角色的阅读指南

---

## 发布步骤

1. 完成上述检查项并修复问题
2. 删除旧文档备份：`rm -rf content-old-backup`
3. 提交 Git 变更
4. 通知团队并收集反馈

---

## 维护计划

| 频率 | 事项 |
|------|------|
| 每周 | 更新项目状态和文档日期 |
| 每月 | 检查文档与代码一致性 |
| 每季度 | 全面审查文档结构和内容 |

**质量指标**：完整性 >=90%，准确性 >=95%，时效性 <=30 天
