# 钱包登录配置

## 技术栈

- RainbowKit v2.2.10 + Wagmi v2.19.5 + viem
- Supabase（对话持久化）
- 双配置模式（SSR / 客户端）

## 核心架构

```
providers.tsx → config.ts → WalletConnectButton.tsx
                    ↓
            page.tsx → supabase/client.ts → conversations.ts
```

## 双配置模式

| 环境 | 连接器 | 原因 |
|------|--------|------|
| SSR | 仅 Injected | 避免 IndexedDB 访问错误 |
| 客户端 | WalletConnect + Injected | 完整钱包连接体验 |

环境检测：`typeof window === 'undefined'` 判断 SSR。

## 钱包连接流程

1. 用户点击连接 → RainbowKit 弹窗
2. 选择钱包 → Wagmi 初始化连接器
3. 连接成功 → 设置钱包上下文
4. 加载/创建对话 → 渲染历史消息

## 对话持久化

- Supabase RLS 策略按 `wallet_address` 隔离数据
- 首次连接自动获取或创建对话
- 消息通过 SSE 流式接收后自动保存云端

## 环境变量

```env
NEXT_PUBLIC_SUPABASE_URL=
NEXT_PUBLIC_SUPABASE_ANON_KEY=
NEXT_PUBLIC_WALLETCONNECT_PROJECT_ID=
```

## 故障排除

| 问题 | 解决方案 |
|------|---------|
| Supabase 初始化失败 | 检查 `.env.local` 中 Supabase 配置 |
| RainbowKit 连接异常 | 确认 WalletConnect Project ID |
| SSR IndexedDB 错误 | 确认双配置模式生效 |
| 刷新后状态丢失 | 检查 cookieStorage 配置 |
| 历史消息无法加载 | 确认 Supabase init.sql 已执行 |
