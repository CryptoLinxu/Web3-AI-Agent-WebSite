# 快速上手指南

## 环境要求

- Node.js >= 18
- pnpm >= 8

## 安装与启动

```bash
pnpm install          # 安装所有依赖
pnpm dev              # 启动开发服务器
```

配置环境变量：复制 `.env.local.example` 为 `.env.local` 并填入 API Key。

## 项目结构

```
├── apps/web/          # Next.js Web 应用
├── packages/
│   ├── ai-config/     # AI 配置包
│   └── web3-tools/    # Web3 工具包
├── skills/x-ray/      # 技能系统
└── docs/              # 项目文档
```

Monorepo 使用 pnpm workspace + turbo 管理。

## 技能系统使用

统一入口：`/origin + 任务描述`

### 任务类型与路由

| 类型 | 说明 | 流程 |
|------|------|------|
| DISCOVER | 探索 | explore |
| BOOTSTRAP | 初始化 | init-docs → update-map |
| DEFINE | 定义 | pm/prd/req → check-in |
| DELIVER-FEAT | 新功能 | 完整链路 |
| DELIVER-PATCH | 修复 | 短链路 |
| DELIVER-REFACTOR | 重构 | 中等链路 |
| VERIFY/GOVERN | 验证治理 | qa/audit/digest |

### 常用命令

```
/origin        总入口
/pipeline      交付管线（feat/patch/refactor）
/explore       只读探索
/check-in      实施前对齐
/coder         编码实现
/digest        复盘沉淀
```

## 学习路径（按周）

| 周 | 主题 | 产出 |
|----|------|------|
| 1 | LLM API、Chat、Function Calling | 最小工具调用闭环 |
| 2 | Agent Loop、Prompt 设计、Web3 工具 | 真实数据查询能力 |
| 3 | Memory、RAG、产品边界 | 稳定系统 Prompt |
| 4 | 工程化、部署、项目表达 | 在线可访问版本 |

## 故障排除

- **依赖安装失败**：清理 node_modules 后 `pnpm install`
- **技能执行异常**：检查是否先完成 check-in
- **文档冲突**：使用 `/resolve-doc-conflicts`
