# 整体架构

## 系统分层

```
用户层 → API层 → Agent核心层 → AI配置层 → 工具层 → 数据层
```

| 层次 | 组件 | 职责 |
|------|------|------|
| 用户层 | apps/web (Next.js) | 聊天界面、钱包连接 |
| API层 | /api/chat, /api/tools, /api/health | 请求路由 |
| Agent核心 | Intent Classifier, Agent Loop, Memory | 意图识别、主循环、会话管理 |
| AI配置 | LLM Factory, OpenAI/Anthropic Adapter | 多模型支持、工厂模式 |
| 提示管理 | prompts.ts, PromptSelectorModal | 提示词配置与选择 |
| 工具层 | packages/web3-tools | 价格/余额/Gas 查询 |
| 数据层 | OpenAI API, Ethereum RPC, 价格API | 外部服务 |

## 技能系统五层结构

| 层次 | 技能 | 职责 |
|------|------|------|
| 入口层 | origin, pipeline | 任务识别与分流 |
| 定义层 | pm, prd, req, check-in | 需求定义与对齐 |
| 交付层 | architect, qa, coder, audit | 设计→验证→实现→审计 |
| 治理层 | digest, update-map | 沉淀与状态更新 |
| 辅助层 | explore, init-docs, browser-verify, resolve-conflicts | 只读探索与治理 |

## 三类交付 Pipeline

| 类型 | 默认链路 |
|------|---------|
| FEAT | pm(按需) → prd → req → check-in → architect → qa → coder → audit → digest → update-map |
| PATCH | req → check-in → coder → qa → digest → update-map |
| REFACTOR | req → check-in → architect → qa → coder → audit → digest → update-map |

## 多模型支持

- **工厂模式**：LLMFactory 管理 Provider 实例，支持缓存
- **OpenAI**：GPT-4/GPT-3.5，完整 Tool Calling + chatStream
- **Anthropic**：Claude 系列，适配消息格式 + chatStream
- **配置驱动**：通过 `DEFAULT_MODEL_PROVIDER` 环境变量切换

## 多数据源容错

策略：顺序尝试多个数据源，单个失败不阻塞，10s 超时。

| 工具 | 数据源 |
|------|--------|
| ETH 价格 | Binance CN → Huobi |
| 钱包余额 | Alchemy → Infura |
| Gas 价格 | 多 RPC 节点 |

## 代理支持

- 自动检测 `HTTP_PROXY` / `HTTPS_PROXY` 环境变量
- 使用 `https-proxy-agent` 创建代理连接
- 仅在配置时启用，支持连接复用

## 质量保证

- QA：FEAT 先 RED 模式，PATCH/REFACTOR 轻量验证
- Coder：最多 10 轮自愈，超限输出 STUCK 报告
- Audit：>=80 通过，60-79 软拒绝，<60 直接拒绝；严重安全问题一票否决
