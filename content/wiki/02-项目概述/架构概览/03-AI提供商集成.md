# AI 提供商集成

## 支持的提供商

| 提供商 | 模型 | 特性 |
|--------|------|------|
| OpenAI | GPT-4, GPT-3.5 | Tool Calling + chatStream |
| Anthropic | Claude 系列 | 适配消息格式 + chatStream |

## 核心架构

```typescript
interface ILLMProvider {
  name: string
  chat(messages, options): Promise<ChatResponse>
  chatStream(messages, options): AsyncGenerator<StreamChunk>
}
```

**LLMFactory** 管理 Provider 实例，支持注册、缓存和按需获取。

## 配置方式

```env
DEFAULT_MODEL_PROVIDER=openai    # 或 anthropic
OPENAI_API_KEY=sk-...
OPENAI_MODEL=gpt-4
ANTHROPIC_API_KEY=sk-ant-...
ANTHROPIC_MODEL=claude-3-sonnet
```

## SSE 流式聊天

### 数据类型

```typescript
type StreamChunk = {
  type: 'content' | 'tool_call' | 'done' | 'error'
  content?: string
  toolCall?: ToolCall
  error?: string
}
```

### 流程

1. 前端 `useChatStream` Hook 发起 POST（Accept: text/event-stream）
2. 后端调用 Provider.chatStream() 获取 AsyncGenerator
3. 构建 ReadableStream 逐块推送 SSE 事件
4. 前端解析 SSE 事件，节流更新 UI（50ms）
5. 遇到 tool_call 时执行工具，带结果再次流式调用

### 前端 Hook 特性

- 内容缓冲 + 节流更新（减少重渲染）
- AbortController 支持取消
- 自动重试（MAX_RETRIES 次）
- 工具调用状态实时跟踪

## 代理支持

自动检测 `HTTPS_PROXY` / `HTTP_PROXY` 环境变量，使用 `https-proxy-agent` 代理外部请求。

## 故障排除

| 问题 | 解决方案 |
|------|---------|
| API Key 未配置 | 检查对应环境变量 |
| 代理连接失败 | 检查代理 URL 和凭据 |
| 流式连接中断 | 检查网络，启用自动重试 |
| 工具调用超时 | 增加超时或使用备用 API |
