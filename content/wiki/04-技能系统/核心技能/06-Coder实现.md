# Coder 编码实现技能

**职责**：在边界清晰的前提下实施代码，通过最多 10 轮自愈循环把 QA 红灯全部变为绿灯。

## 输入/输出

- **输入**：check-in 边界与标准、architect 架构说明、qa 测试清单与红灯结果
- **输出**：可验证通过的代码实现；或 STUCK 报告（超 10 轮时）

## 自愈循环机制

每轮步骤：实施代码 -> 运行相关验证 -> 读取失败信息 -> 根因分析 -> 修复 -> 下一轮

| 规则 | 说明 |
|------|------|
| 最多 10 轮 | 超过 10 轮立即终止 |
| 终止后输出 | 卡住原因、已尝试方案、当前阻塞点、建议人工介入方向 |
| 优先跑相关验证 | 避免全量重跑 |

## 核心规则

| 规则 | 说明 |
|------|------|
| 不修改 docs 需求定义 | 不擅自修改验收标准 |
| 不跳过失败验证 | 必须全部 GREEN |
| 红灯与需求矛盾时 | 停止并报告，不自行修改需求 |
| 必须先完成 check-in | 否则不得进入 coder |

## 衔接关系

- **上游**：qa（提供红灯结果与验证清单）
- **下游**：audit（交付实现结果进行风险审计）
- **红绿衔接**：FEAT 时 QA 先 RED，Coder 将 RED 全部变为 GREEN
