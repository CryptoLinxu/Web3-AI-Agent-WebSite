# Resolve-Doc-Conflicts 文档冲突解决技能

**职责**：专门处理文档层面的合并冲突与内容一致性问题，避免将文档治理与代码修复混在一起。

## 触发场景

- 文档合并冲突
- 索引或地图冲突
- trace/审计/规划类文档冲突
- VERIFY/GOVERN 任务中的文档治理

## 输入/输出

- **输入**：冲突中的文档文件
- **输出**：已解决的文档冲突、需要人工复核的冲突点

## 执行流程

1. 扫描冲突文件
2. 区分生成物与手写文档
3. 生成物优先重建
4. 手写内容优先保留信息，不盲猜
5. 输出待人工确认项

## 核心规则

| 规则 | 说明 |
|------|------|
| 不处理业务代码冲突 | 仅处理文档 |
| 能保留两边内容时优先保留 | 信息优先策略 |
| 无法安全判断时显式标记人工介入 | 不假装理解模糊语义 |
| 与 qa/audit/digest/update-map 协同 | 形成文档治理闭环 |
