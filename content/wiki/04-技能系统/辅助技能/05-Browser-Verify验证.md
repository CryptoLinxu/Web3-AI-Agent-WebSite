# Browser-Verify 浏览器验证技能

**职责**：浏览器层的可视化与交互验收，主要面向前端页面、交互流程与页面回归验证。

## 触发场景

- 前端页面验收
- 交互流程验证
- 可视回归检查
- PATCH 的浏览器级复验

## 输入/输出

- **输入**：目标页面或入口、修复/改动说明、验证步骤
- **输出**：结果（PASS / PARTIAL / FAIL）、发现的问题

## 执行流程

1. 进入目标页面
2. 执行验证步骤
3. 记录关键结果
4. 检查可视和交互回归

## 核心规则

| 规则 | 说明 |
|------|------|
| 不直接改代码 | 仅验收，不实施 |
| 不代替单元/集成测试 | 强调浏览器级复验 |
| 按需插入 | 可用于 FEAT/PATCH/REFACTOR，但不每次强制 |
| 前端行为需肉眼确认时优先使用 | 核心判断标准 |
| 通过回 audit/closeout | 失败回退 coder |
