# Update-Map 地图更新技能

**职责**：维护项目状态、索引与下一步入口建议，确保下一轮任务能基于最新上下文继续推进。

## 触发场景

- 交付型任务完成后（digest 之后）
- BOOTSTRAP 完成后（init-docs 之后）
- VERIFY/GOVERN 完成后

## 输入/输出

- **输入**：digest 沉淀内容、当前项目状态
- **输出**：更新后的地图与索引、下一步建议、变更日志（自动调用 changelog）

## 核心规则

| 规则 | 说明 |
|------|------|
| 贯穿全流程 | 从探索到交付到治理都可能触发 |
| 与 digest 职责分离 | digest 负责经验沉淀，update-map 负责状态更新 |
| 自动调用 changelog | 交付型任务完成后自动生成变更日志 |
| 返回 origin | 更新完成后回到主入口，准备下一轮 |
