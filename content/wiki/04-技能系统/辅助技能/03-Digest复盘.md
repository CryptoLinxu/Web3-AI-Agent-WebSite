# Digest 复盘沉淀技能

**职责**：在任务完成后进行阶段复盘与知识沉淀，聚焦"经验与教训"的提炼，而非流水账式的变更记录。

## 触发场景

- 交付型任务链路末端（FEAT/PATCH/REFACTOR 完成后）
- audit 通过后自动进入

## 输入/输出

- **输入**：本轮产物、QA 结果、Audit 结论
- **输出**：结构化复盘报告
  - 本轮完成了什么
  - 遇到了什么问题
  - 学到了什么
  - 仍未解决的问题
  - 下一步建议

## 核心规则

| 规则 | 说明 |
|------|------|
| 重点记录"为什么卡住/为什么成功" | 避免流水账 |
| PATCH 可轻量写 | 但不建议省略 |
| 不代替 update-map | 仅承担经验沉淀 |
| 不替代 PRD | 不做需求定义 |
| 完成后进入 update-map | 确保状态同步 |
