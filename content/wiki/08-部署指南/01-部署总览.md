# 部署总览

## 系统要求

| 项目 | 最低配置 | 推荐（生产） |
|------|----------|-------------|
| Node.js | >= 18.0.0 | 18.x LTS |
| pnpm | >= 8.0.0 | 8.15.0 |
| 内存 | 2GB | 4GB+ |
| 磁盘 | 1GB | 5GB+ |

---

## 部署方案对比

| 方案 | 适用场景 | 复杂度 | 成本 |
|------|----------|--------|------|
| **Vercel** (推荐) | 快速上线、个人项目 | 低 | 免费额度 |
| **Docker** | 私有化部署、自定义环境 | 中 | 中等 |
| **传统服务器** | 企业级部署、完全控制 | 高 | 高 |

---

## 方案一：Vercel 部署

**构建设置**：
```
Framework: Next.js
Root Directory: apps/web
Build Command: pnpm build
Install Command: pnpm install --frozen-lockfile
Output Directory: .next
```

必须开启：**Include files outside the root directory in the Build Step**

**步骤**：推送到 GitHub -> 导入 Vercel 项目 -> 配置环境变量 -> Deploy

---

## 方案二：Docker 部署

```bash
docker-compose build && docker-compose up -d
```

使用多阶段构建（base -> deps -> builder -> runner），非 root 用户运行，端口 3000。

---

## 方案三：传统服务器

```bash
git clone ... && pnpm install --frozen-lockfile && pnpm build
```

使用 PM2 管理进程 + Nginx 反向代理 + Let's Encrypt HTTPS。

Nginx 关键配置：`proxy_buffering off`（SSE 流式支持）。

---

## 环境变量

### 必需

| 变量名 | 说明 |
|--------|------|
| `NEXT_PUBLIC_WALLETCONNECT_PROJECT_ID` | WalletConnect 项目 ID |
| `NEXT_PUBLIC_SUPABASE_URL` | Supabase 项目 URL |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | Supabase 匿名 Key |

### 可选

| 变量名 | 说明 | 默认值 |
|--------|------|--------|
| `HTTPS_PROXY` | HTTP 代理（国内需要） | - |
| `APP_VERSION` | 应用版本 | 0.1.0 |
| `SUPABASE_SERVICE_ROLE_KEY` | 服务端特权密钥（生产必需） | - |

---

## 数据库配置（Supabase）

**数据表**：conversations（对话）、messages（消息）、transfer_cards（转账卡片）

**迁移执行**：
```bash
supabase db push
# 或手动执行：init.sql -> create_transfer_cards.sql -> upgrade_production_rls.sql
```

**RLS 策略**（生产环境）：
- SELECT/INSERT/UPDATE：应用层过滤 + RLS 双重保护
- DELETE：仅服务端 API 执行（前端直接调用被 RLS 拦截）

---

## 部署检查清单

- [ ] 环境变量全部配置
