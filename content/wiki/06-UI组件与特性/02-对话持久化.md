# 对话持久化系统

## 概述

基于 Supabase 的对话持久化系统，结合钱包身份验证实现对话历史的安全存储与管理。支持懒加载创建、级联删除和行级权限控制(RLS)。

## 数据模型

**数据库表结构** (Supabase PostgreSQL):

| 表名 | 主要字段 | 说明 |
|------|---------|------|
| conversations | id, wallet_address, title, created_at, updated_at | 对话记录，RLS按wallet_address |
| messages | id, conversation_id(FK), role, content, tool_calls, created_at | 消息记录，级联删除 |
| transfer_cards | id, conversation_id(FK), message_id, from, to, token_symbol, amount, chain, status, tx_hash | 转账卡片状态 |

**外键约束**: messages/transfer_cards 通过 `ON DELETE CASCADE` 关联 conversations，删除对话时自动清理。

## 核心API

### 对话服务 (`/lib/supabase/conversations.ts`)

```typescript
// 核心函数
getConversations(walletAddress): Conversation[]      // 获取对话列表
getLatestConversation(walletAddress): Conversation   // 懒加载：获取最新对话
createConversation(walletAddress): Conversation      // 创建新对话
saveMessage(conversationId, message): Message        // 保存消息
loadMessages(conversationId): Message[]              // 加载历史消息
deleteConversation(id, walletAddress): void          // 单步删除(级联)
verifyOwnership(conversationId, walletAddress): bool // 所有权验证
```

### 转账服务 (`/lib/supabase/transfers.ts`)

```typescript
createTransferCard(data): TransferCard    // 创建转账卡片
updateTransferStatus(id, status, txHash): void  // 更新转账状态
getTransferCards(conversationId): TransferCard[]  // 获取转账卡片列表
```

### REST API端点

| 端点 | 方法 | 功能 |
|------|------|------|
| /api/chat | POST | 聊天+消息持久化 |
| /api/supabase/delete-conversation | DELETE | 单步删除对话 |
| /api/supabase/verify-ownership | POST | 验证对话所有权 |

## 关键机制

**懒加载优化**: 不在钱包连接时自动创建对话，而是延迟到第一条AI消息发送时才创建数据库记录。

**对话生命周期**:
1. 用户连接钱包 -> `setWalletContext(address)`
2. 加载历史对话列表 -> `getConversations(address)`
3. 发送第一条消息时 -> `createConversation(address)` (懒加载)
4. 每轮对话 -> `saveMessage()` 持久化
5. 删除对话 -> 单步级联删除所有关联数据

**安全机制**: Supabase RLS 策略确保用户只能访问自己钱包地址的对话数据。生产环境需配合 Supabase Auth + JWT。
