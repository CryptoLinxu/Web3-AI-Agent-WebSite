# 内存管理系统

## 概述

管理对话历史和上下文的策略系统，采用 Strategy 模式设计。提供 L3 摘要压缩和 L2 滑动窗口两种策略，为 L4 向量数据库扩展预留接口。

## MemoryManager 接口

```typescript
interface MemoryManager {
  addMessage(message: Message): void
  getMessages(): Message[]
  shouldCompress(): boolean
  compress(): Promise<void>
  clear(): void
}

interface MemoryConfig {
  compressThreshold: number   // 触发压缩的消息数阈值（默认10）
  keepRecentCount: number     // 保留的最近消息数（默认5）
  summaryModel?: string       // 摘要用模型（可选）
}

type MemoryStrategy = 'l3-compression' | 'l2-sliding-window'
```

## 策略对比

| 特性 | L3 摘要压缩 | L2 滑动窗口 |
|------|------------|------------|
| 原理 | 定期将早期消息压缩为摘要 | 保留最近N条，丢弃其余 |
| Token消耗 | 降低>=50% | 与对话长度线性增长 |
| 额外LLM调用 | 需要（生成摘要） | 不需要 |
| 历史信息 | 通过摘要保留关键信息 | 完全丢失早期历史 |
| 响应延迟 | 压缩期间有延迟 | 无延迟 |
| 适用场景 | 长对话、复杂交互 | 短对话、简单交互 |

## L3 摘要压缩核心逻辑

1. `addMessage()` 添加消息到 `originalMessages`
2. 检查是否达到 `compressThreshold` 且未在压缩中
3. 异步调用 LLM 生成摘要（`isCompressing` 标志防并发）
4. 压缩完成后更新 `summary` 字段，裁剪已压缩消息
5. `getMessages()` 返回: `[summary作为system消息, ...最近N条原始消息]`

**错误处理**: 压缩失败时保留完整历史，下次重试。

## L2 滑动窗口核心逻辑

- `addMessage()` 追加到 `messages` 数组
- `getMessages()` 返回 `messages.slice(-windowSize)`
- `shouldCompress()` 始终返回 `false`

## 配置方式

通过环境变量覆盖默认配置:
- `NEXT_PUBLIC_MEMORY_COMPRESS_THRESHOLD`: 压缩阈值
- `NEXT_PUBLIC_MEMORY_KEEP_RECENT`: 保留消息数

支持运行时动态切换策略，切换时创建新实例，立即生效。
